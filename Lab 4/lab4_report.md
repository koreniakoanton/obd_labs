Лабораторна робота 4: Аналітичні SQL-запити (OLAP)
Тема проєкту: Медична інформаційна система для управління прийомами та діагнозами.

У цьому файлі наведено опис аналітичних запитів, створених для аналізу активності пацієнтів, навантаження на лікарів та статистики лікування.

1. Агрегатні функції та групування (GROUP BY, HAVING)
Ці запити використовуються для підбиття підсумків та отримання зведеної статистики по клініці:

Запит 1.1: Використовує функцію COUNT для підрахунку загальної кількості медичного персоналу в системі та MAX, щоб знайти дату останнього запланованого прийому. Це дозволяє оцінити поточний штат та горизонт планування.

Запит 1.2: Використовує функцію COUNT разом із GROUP BY для обчислення кількості прийомів у розрізі кожного лікаря. Це допомагає зрозуміти реальне навантаження на різних спеціалістів.

Запит 1.3: Використовує функцію COUNT та речення HAVING для фільтрації згрупованих даних. Запит виводить лише тих лікарів, які є найбільш активними (мають більше одного запису на прийом).

Запит 1.4: Використовує функцію COUNT для підрахунку загальної кількості виданих рецептів, що дозволяє оцінити інтенсивність призначеного лікування.

2. Об'єднання таблиць (JOIN)
Ці запити об'єднують дані з кількох нормалізованих таблиць для формування зрозумілих звітів для адміністрації клініки:

Запит 2.1 (INNER JOIN): Об'єднує таблиці appointments, patients та doctors. Замість незрозумілих ідентифікаторів (ID), запит формує детальний розклад, виводячи ПІБ пацієнта, спеціалізацію лікаря та точний час візиту.

Запит 2.2 (LEFT JOIN): Об'єднує patients та appointments. Використання LEFT JOIN гарантує, що у звіті з'являться абсолютно всі пацієнти з бази даних, навіть ті, хто ще не записувався на прийом (наприклад, новостворені картки).

Запит 2.3 (RIGHT JOIN): Об'єднує appointments та doctors. Дозволяє побачити повний список лікарів і прийоми, які вони проводять. RIGHT JOIN гарантує виведення лікарів-"сиріт", до яких на даний момент ще немає жодного запису.

3. Підзапити (Subqueries)
Підзапити дозволяють використовувати результати аналізу одного рівня як умову для складнішої вибірки:

Запит 3.1 (Підзапит у WHERE): Використовується для глибокої фільтрації. Основний запит виводить дані пацієнтів, а вкладений підзапит крок за кроком знаходить їх через ланцюжок "Рецепт -> Діагноз -> Прийом", щоб відібрати лише тих, кому було поставлено конкретний діагноз (наприклад, "Гіпертонія").

Запит 3.2 (Підзапит у SELECT): Для кожного лікаря виконується вкладений підзапит, який прямо у стовпці результату підраховує кількість його візитів, виводячи персональний показник "ефективності" поруч із його прізвищем.

Запит 3.3 (Підзапит з IN): Підзапит створює динамічний список diagnosis_id, для яких існують виписані рецепти, а основний запит виводить дані лише про ті прийоми, які закінчилися реальним медикаментозним призначенням.
